# Nexus Coach Language & Grammar

> Consistent writing conventions that build trust through predictable, professional communication.

## Strategic Foundation

**Brand Alignment:** Nexus Coach's language and grammar conventions reflect the Clarity and Professionalism values. Consistent conventions reduce cognitive load — coaches learn the patterns once and trust their understanding throughout the platform. Like a well-organized practice, our writing follows clear rules so the reader never has to guess.

## Capitalization

### Sentence Case (Default)
Use sentence case for all UI text unless specified otherwise. Sentence case is more readable, more human, and more aligned with our warm-but-professional voice.

| Element | Style | Example |
|---------|-------|---------|
| Page titles | Sentence case | "Client overview" |
| Section headings | Sentence case | "Upcoming sessions" |
| Button labels | Sentence case | "Schedule session" |
| Menu items | Sentence case | "View session notes" |
| Tab labels | Sentence case | "Active clients" |
| Form labels | Sentence case | "Email address" |
| Column headers | Sentence case | "Last session date" |
| Placeholder text | Sentence case | "Search clients..." |

### Title Case Exceptions
Use title case only for:
- The product name: "Nexus Coach"
- Proper nouns: "Sarah Chen," "ICF Certification"
- Feature names when they're marketed as named products (e.g., "Practice Insights")

### Uppercase Exceptions
Use uppercase only for:
- Sidebar section labels (e.g., "PRACTICE" / "INSIGHTS")
- Abbreviations (e.g., "ICF," "CTA," "API")

## Punctuation

### Oxford Comma
Always use the Oxford comma in lists of three or more items.

```
✓ Coaches, practice leads, and clients
✗ Coaches, practice leads and clients
```

### Periods
- **Use periods** in complete sentences, descriptions, help text, and body content
- **Omit periods** in headings, button labels, navigation items, placeholder text, and single-phrase descriptions

### Colons
Use colons to introduce related information. Capitalize the first word after a colon only if it starts a complete sentence.

```
✓ Session status: In progress
✓ Note: This action can't be undone.
```

### Dashes
- **Em dash (—):** For emphasis or parenthetical breaks. No spaces on either side.
  ```
  "Every detail — from scheduling to billing — is handled."
  ```
- **En dash (–):** For ranges (dates, times, numbers).
  ```
  "Monday–Friday" / "9:00 AM–5:00 PM" / "10–15 clients"
  ```

### Exclamation Points
Use sparingly — maximum one per screen. Reserve for genuine celebration or positive moments.

```
✓ "Welcome to Nexus Coach!"
✓ "Program complete — congratulations!"
✗ "Your settings have been saved!"
✗ "Error! Something went wrong!"
```

### Quotation Marks
Use double quotes for direct quotations. Use single quotes for quotes within quotes. Place commas and periods inside quotes.

## Abbreviations & Acronyms

### Common Abbreviations

| Abbreviation | Full Form | When to Use |
|-------------|-----------|-------------|
| ICF | International Coaching Federation | Spell out on first use, abbreviate after |
| CTA | Call to action | Internal only — never in UI |
| API | Application programming interface | Developer documentation only |
| CSV | Comma-separated values | File export contexts |
| PDF | Portable document format | File export contexts |

### Rules
- Spell out abbreviations on first use within a page, followed by the abbreviation in parentheses
- Common abbreviations readers already know (PDF, CSV) don't need expansion
- Never abbreviate coaching-specific terms — always write "session," "program," "practice" in full

## Numbers

### Formatting Rules

| Context | Rule | Example |
|---------|------|---------|
| One through nine | Spell out | "three sessions" |
| 10 and above | Use numerals | "12 clients" |
| Starting a sentence | Always spell out | "Twenty-four hours remain" |
| Mixed in same sentence | Use numerals for all | "2 coaches and 15 clients" |
| Percentages | Numeral + symbol | "85% engagement" |
| Currency | Symbol + numeral | "$150.00" |
| Large numbers | Comma-separated | "1,250 sessions" |
| Phone numbers | Standard format | "(555) 123-4567" |
| Ranges | En dash, no spaces | "10–15 clients" |

### Ordinals
Spell out ordinals: "first session," "third program." Avoid superscript (1st, 2nd).

## Contractions

Contractions are encouraged — they make content feel warm and conversational, aligning with The Guide's approachable voice.

| Use | Instead of |
|-----|-----------|
| "You're all set" | "You are all set" |
| "We couldn't save" | "We could not save" |
| "That's 12 sessions" | "That is 12 sessions" |
| "Don't worry" | "Do not worry" |

### When to Avoid Contractions
- Legal content, terms of service, or privacy policies
- When emphasis requires the full form: "This action *cannot* be undone"
- Content that will be translated (contractions complicate localization)

## Lists

### Bullet Lists
- Use for items with no specific order
- Keep items parallel in structure (all nouns, all phrases, or all sentences)
- If items are fragments, omit end punctuation
- If items are complete sentences, use end punctuation on all of them

### Numbered Lists
Use when order matters — steps, processes, ranked items.

### List Capitalization
Capitalize the first word of each list item, regardless of whether it's a sentence or fragment.

## Parallel Structure

Keep related items grammatically consistent.

```
✓ "Schedule sessions, track progress, and manage billing"
✗ "Schedule sessions, tracking of progress, and you can manage billing"

✓ Navigation: Dashboard, Clients, Sessions, Programs
✗ Navigation: Dashboard, View Clients, Sessions, My Programs
```

## Active Voice

Prefer active voice — it's direct, clear, and empowering.

```
✓ "You scheduled a session with Sarah Chen."
✗ "A session with Sarah Chen has been scheduled."

✓ "We couldn't save your changes."
✗ "Your changes could not be saved by the system."
```

Use passive voice only when the actor is unknown or irrelevant:
```
✓ "Your invoice was viewed on January 15." (The client viewed it — passive focuses on the invoice)
```

## Localization Considerations

For future internationalization:
- Avoid idioms, slang, or culturally specific references
- Don't hardcode date/time formats (see Date & Time guidelines)
- Avoid contractions in content intended for translation
- Use complete sentences rather than fragments that rely on word order
- Keep UI text short — translated text can be 30-50% longer

## Usage Guidelines

### Do
- Apply these conventions consistently across all platform content
- Use sentence case as the default for everything
- Keep lists parallel in structure
- Write in active voice
- Use contractions for warmth

### Don't
- Capitalize words for emphasis (use bold or structure instead)
- Use abbreviations the user wouldn't know
- Mix capitalization styles within the same screen
- Write sentence fragments where complete sentences aid understanding
- Use slang, idioms, or overly casual language
