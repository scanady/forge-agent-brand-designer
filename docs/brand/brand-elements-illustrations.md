# OpenLife Illustrations

> Visuals that convey complex ideas with clarity and personality.

## Strategic Foundation

**Brand Alignment:** OpenLife illustrations explain concepts that photography cannot capture. They embody our values of clarity and openness—simple enough to understand quickly, distinctive enough to be recognizable. Illustrations support The Creator archetype, visualizing innovation and possibility.

## Illustration Style

### Approach
Abstract and conceptual over literal representation.

| Attribute | Direction |
|-----------|-----------|
| Style | Clean, geometric, modern |
| Complexity | Simple, essential elements |
| Geometry | Confident lines, soft curves |
| Depth | Subtle shadows, layered |
| Figures | Stylized, diverse silhouettes |

### Style Characteristics
- Confident, clean lines
- Balanced geometric shapes
- Purposeful negative space
- Subtle depth and dimension
- Cohesive with icon system

## Color Palette

### Primary Illustration Colors

| Color | HEX | Usage |
|-------|-----|-------|
| Liberation Blue | `#0066CC` | Primary elements |
| Collaboration Teal | `#0891B2` | Secondary elements |
| Innovation Indigo | `#4F46E5` | Accent, technology |
| Transform Orange | `#EA580C` | Highlights, emphasis |

### Supporting Colors
- Gray 200–400 for backgrounds and subtle elements
- White for negative space and contrast
- Foundation Navy for anchoring elements

### Color Guidelines
- Use brand palette exclusively
- Limit to 2-3 colors per illustration
- Maintain visual hierarchy with color
- Ensure accessibility contrast

## Illustration Types

### Spot Illustrations
Small, focused graphics for specific concepts.

| Attribute | Specification |
|-----------|---------------|
| Size | 64–200px |
| Complexity | Single concept |
| Usage | Feature callouts, process steps |

### Hero Illustrations
Larger, more complex compositions.

| Attribute | Specification |
|-----------|---------------|
| Size | Full-width or major section |
| Complexity | Multiple elements |
| Usage | Landing pages, section headers |

### Diagrams
Explanatory graphics for processes or architecture.

| Attribute | Specification |
|-----------|---------------|
| Size | Context-dependent |
| Elements | Clear labels, annotations |
| Usage | Documentation, technical content |

## Character Style

If depicting people:
- Simplified, stylized silhouettes
- Diverse representation
- No detailed facial features
- Consistent proportions
- Professional contexts

### Functional

### Marketing
- Hero sections

## Complexity Levels

| Level | Elements | Usage |
|-------|----------|-------|
| Simple | 2-3 shapes | Icons, small spots |
| Medium | 4-8 elements | Feature illustrations |
| Complex | 8+ elements | Hero, conceptual |

## Technical Specifications

### File Formats
- SVG for web (scalable)
- PNG for fallbacks (2x resolution)
- PDF for print

### Dimensions
- Design at 2x for retina
- Provide multiple sizes when needed
- Maintain aspect ratios

## Accessibility

- Don't rely on illustrations alone for information
- Provide alt text describing the concept
- Ensure sufficient contrast
- Test with colorblind simulators

## Usage Guidelines

### Do
- Use brand colors exclusively
- Keep illustrations purposeful
- Maintain consistent style
- Include diverse representation
- Pair with supporting text

### Don't
- Use photorealistic styles
- Create overly complex graphics
- Mix illustration styles
- Use dated visual metaphors
- Overcrowd compositions

## Examples

### Empty State

```
Illustration: Abstract representation of documents/data
Purpose: Indicate no data yet, encourage action
Elements: Simple geometric shapes, subtle blue tones
Text: "No policies yet. Create your first policy to get started."
```

### Feature Highlight

```
Illustration: Connected nodes representing community
Purpose: Visualize collaborative platform
Elements: Circles connected by lines, diverse dots
Colors: Liberation Blue primary, Teal secondary
```

### Process Diagram
```
Illustration: Flow from left to right
Purpose: Explain deployment pipeline
Elements: Labeled boxes, arrows, icons
```
