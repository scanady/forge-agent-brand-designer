# iFoundry Media Illustrations

> Clean, technical illustrations that make complex engineering concepts tangible.

## Strategic Foundation

**Brand Alignment:** iFoundry illustrations embody The Builder's pragmatism and The Sage's ability to clarify complexity. They translate architecture patterns, product lifecycles, and engineering workflows into clean, approachable visuals. Illustrations are functional — they explain what words alone can't, reduce cognitive load for technical and non-technical audiences alike, and reinforce iFoundry's identity as a team that makes complex things understandable.

## Illustration Style

### Overall Approach
- **Geometric and angular:** Built from nodes, lines, and geometric shapes — echoing the logo's connected-node motif
- **Flat with technical depth:** Primarily flat design with subtle layering for hierarchy
- **Brand-colored:** Drawn from the iFoundry palette (Forge Blue, Ignite Orange, Iron Slate, cool grays)
- **Structural, not decorative:** Every illustration explains a concept, maps a process, or represents a state
- **Connected:** Visual language of nodes and links runs through illustrations, reinforcing the "forging connections" metaphor

### Visual Characteristics

| Attribute | Value |
|-----------|-------|
| Line weight | 1.5px (consistent with icon stroke) |
| Corner radius | Mix of sharp angles and rounded terminals (matching logo's angular-to-rounded transitions) |
| Color palette | Brand colors at 100% and reduced opacities (10%, 20%, 40%) |
| Shadows | Minimal — soft, brand-colored shadows at 8–12% opacity |
| Backgrounds | Transparent or Slate 50 (`#F8FAFC`) |
| Grid alignment | Aligned to 8px grid |
| Connectors | Angular lines with rounded endpoints (node-to-node style) |

## Illustration Types

### 1. Spot Illustrations
Small, focused illustrations that accompany UI elements and content.

| Attribute | Value |
|-----------|-------|
| Size | 48–120px |
| Complexity | Low — 1–2 elements |
| Color | 2–3 brand colors max |
| Format | SVG |

**Used for:**
- Empty states ("No deployments yet")
- Feature callouts and onboarding
- Status indicators on dashboards
- List item decorations

**Examples:**
- A node cluster forming into a unified shape (product taking form)
- A rocket departing a launch pad (deployment complete)
- Connected gears with a checkmark (integration successful)
- A blueprint transforming into a live screen (concept to product)

### 2. Scene Illustrations
Larger narrative illustrations that explain concepts, features, or workflows.

| Attribute | Value |
|-----------|-------|
| Size | 200–600px |
| Complexity | Medium — 3–5 elements |
| Color | Full brand palette |
| Format | SVG or PNG @2x |

**Used for:**
- Website landing page features
- Case study headers
- Service page hero areas
- Documentation section headers

**Examples:**
- Raw material (scattered nodes) flowing into a foundry (process) and emerging as a polished product
- A team of simplified figures connected around a central product architecture
- The product lifecycle: Concept → Architecture → Build → Launch → Scale
- Multiple technology stacks converging into a unified platform

### 3. Diagram Illustrations
Technical visual explanations of workflows, architectures, and system relationships.

| Attribute | Value |
|-----------|-------|
| Size | 300–800px |
| Complexity | High — multiple connected elements |
| Color | Full brand palette with connector lines |
| Format | SVG |

**Used for:**
- Architecture documentation
- Service offering breakdowns
- Product lifecycle flowcharts
- Technical proposal visuals

**Examples:**
- Microservices architecture: Independent services connected through an event bus
- Deployment pipeline: Code → Build → Test → Stage → Production
- iFoundry engagement model: Discovery → Architecture → Sprint → Launch → Scale
- Data flow diagrams with clear inputs, transformations, and outputs

## Character Style

Human figures in iFoundry illustrations are abstract, technical, and inclusive.

### Approach
- **Geometric silhouettes:** Clean shapes, no facial features — consistent with the technical aesthetic
- **Brand-colored figures:** Use Forge Blue, Slate, and Ignite Orange — not skin tones
- **Role through context:** Differentiate engineers, architects, founders through tools in hand (keyboard, whiteboard marker, presentation screen) rather than appearance
- **Active postures:** Convey energy through body language — leaning into screens, presenting, building
- **Small scale:** Figures serve the diagram, not the other way around

### Size Guidelines

| Context | Height | Detail Level |
|---------|--------|-------------|
| Spot illustration | 24–48px | Abstract shape only |
| Scene illustration | 60–120px | Simplified body, posture visible |
| Diagram | 32–64px | Icon-like, role-differentiated |

## Complexity Levels

### Level 1 — Simple
Single element, 2 colors, no background. For in-app micro-moments.

```
Elements: 1–2
Colors: 2 (primary + neutral)
Detail: Minimal
Example: A connected-node cluster (concept forming)
```

### Level 2 — Moderate
Small scene, 3–4 colors, optional background. For features and states.

```
Elements: 3–5
Colors: 3–4 (primary + accent + neutral)
Detail: Moderate — shapes suggest context
Example: A deployment pipeline with build, test, and deploy stages
```

### Level 3 — Detailed
Full scene or workflow, full palette, background elements. For marketing and documentation.

```
Elements: 5+
Colors: Full palette
Detail: Higher — clear narrative or process
Example: End-to-end product lifecycle from idea to scale
```

- 100% — Primary subject elements and key connections
- 40% — Secondary elements that support the narrative
- 20% — Background shapes and containers
- 10% — Subtle fills and atmosphere

## The Node Motif

A distinctive visual element carried through iFoundry illustrations: the **connected node** — circular dots linked by angular lines, directly inspired by the logo.

### Usage
- **Process stages** represented as nodes connected by directional lines
- **Architecture diagrams** using nodes for services and lines for data flow
- **Team/partnership** visualized as converging nodes forming a larger structure
- **Product formation** shown as scattered nodes being drawn together into a unified product

### Specifications
- Node diameter: 8px (small), 12px (medium), 16px (large)
- Connector stroke: 1.5px, angular with rounded endpoints
- Connector color: Slate 300 (inactive), Forge Blue (active), Ignite Orange (energy)

## Accessibility

- Illustrations must not be the sole method of conveying critical information
- Provide descriptive `alt` text for all illustrations that convey meaning
- Use `alt=""` with `aria-hidden="true"` for purely decorative illustrations
- Ensure sufficient contrast between illustration elements and their backgrounds
- Avoid color as the only differentiator between elements — use shape and position

## Usage Guidelines

### Do
- Use illustrations to clarify complex engineering and product concepts
- Maintain consistent geometric style with the connected-node motif
- Align illustrations to the 8px grid
- Use brand colors at approved opacity levels
- Pair illustrations with clear text labels or descriptions
- Use the node motif to echo the logo's visual language

### Don't
- Create realistic or photographic-style illustrations
- Use illustrations as pure decoration without communicative purpose
- Mix illustration styles within the same context
- Use colors outside the brand palette
- Add excessive detail that competes with adjacent content
- Use cartoon-style, whimsical, or playful illustrations — the tone is technical and confident
